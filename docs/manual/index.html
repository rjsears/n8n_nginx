n8n Management Suite — User Manual

Comprehensive guide to operating the n8n Management Console after installation. Every UI option, every modal, every sub-tab.

Welcome

This manual is intended for operators who have already installed and deployed the n8n Management Suite. It does not cover initial installation — see the project README for setup, or the migration guide if upgrading from v2.

The audience is the person who logs into the management console day-to-day: starting and stopping containers, running and verifying backups, configuring notifications, troubleshooting SSL, managing access. If that's you, you're in the right place.

How to read this manual

The manual is organized in three parts. Read sequentially, or jump directly to the section you need via the navigation above.

Every page has a left-hand sidebar with anchors to its subsections. Use the prev/next links at the bottom of each page to step through in order.

📝 Note: Screenshots in this manual were captured from a live test deployment. Some screenshots contain sensitive data (API tokens, internal IPs, host names) which are flagged inline and indexed in the Security Flag Index. Blur those images before publishing or sharing this manual outside your organization.

Part 1: User Guide

Everyday operations. The pages most operators will visit most often.

  • Dashboard — system overview at a glance: container health, resource usage, recent activity
  • Containers — start, stop, restart Docker services; inspect logs and stats
  • Flows — view, toggle, and trigger n8n workflows; jump to n8n itself
  • Backups — schedule, run, verify, restore; configure retention and storage
  • Notifications — channels, groups, rules, NTFY topics, n8n webhook integration

Part 2: Administration

System-level configuration and integration management. Less frequent but more consequential.

  • System — health checks, SSL renewal, network, Cloudflare/Tailscale integrations
  • Settings — every configuration knob: NFS, access control, email, environment variables, Redis cache, and more

Part 3: Appendix

Reference material and the master security index.

  • Troubleshooting — symptom-to-section quick reference
  • Security Flag Index — all screenshots requiring blur before publishing

Legend

Throughout the manual, you'll see colored callout boxes. Their meaning is consistent:

📝 Note: additional context, not required reading.
💡 Tip: a recommended approach or shortcut.
⚠️ Warning: behavior that may surprise you, or has caveats.
🚫 Danger: destructive or irreversible action.
🔒 Security flag: screenshot contains sensitive data; blur before publishing.
